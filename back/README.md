# توثيق الجزء الخلفي (Backend Documentation) - EVO

هذا المجلد يحتوي على كود خادم (Server) مشروع **EVO**، والمبني باستخدام **Node.js** و **Express**.

## التقنيات المستخدمة
- **Node.js:** بيئة التشغيل الأساسية.
- **Express.js:** إطار عمل للتعامل مع الـ HTTP Requests والـ Routing.
- **JWT (JSON Web Tokens):** لإدارة المصادقة (Authentication).
- **MySQL / PostgreSQL:** قواعد البيانات المستخدمة (عبر مكتبات `mysql2` و `pg`).
- **Bcryptjs:** لتشفير كلمات المرور.
- **Cors:** للسماح بالطلبات من الواجهة الأمامية (Frontend).

## هيكل المشروع (Folder Structure)

- **`server.js`:** الملف الأساسي ونقطة انطلاق الخادم. يقوم بإعداد Middleware وتحميل المسارات (Routes) تلقائياً.
- **`routes/`:** يحتوي على جميع ملفات المسارات البرمجية (الأندبوينتس).
- **`middleware/`:** يحتوي على وظائف الوسيط (مثل التحقق من التوكين).
- **`config/`:** ملفات إعدادات قاعدة البيانات والاتصال.
- **`utils/`:** أدوات مساعدة وملفات SQL لإنشاء الجداول.
- **`uploads/`:** المجلد المخصص لتخزين الملفات المرفوعة.

## نظام المصادقة (Authentication Flow)

يستخدم النظام توقيتات **JWT** لتأمين المسارات:
1. **Middleware (`middleware/auth.js`):** وظيفة `authenticateToken` تقوم بفحص الهيدر `Authorization` بحثاً عن `Bearer <token>`.
2. في حالة صحة الـ Token، يتم السماح بمرور الطلب وإضافة بيانات المستخدم إلى `req.user`.
3. في حالة الخطأ، يتم إرجاع كود `401` (غير مصرح) أو `403` (التوكين غير صالح).

## المسارات البرمجية (API Routes)

يتم تحميل المسارات ديناميكياً من مجلد `routes/` تحت المسار `/api/<filename>`. أهم المسارات:

- **الطلاب (`/api/students`):** للتعامل مع بيانات الطلاب (إضافة، تعديل، حذف، بحث).
- **المستخدمين (`/api/USER`):** لإدارة حسابات النظام والأدوار.
- **المواد الدراسية (`/api/course`):** للتعامل مع الكورسات والمناهج.
- **الدرجات (`/api/grade`):** لتسجيل وتعديل درجات الطلاب.
- **الأخبار (`/api/news`):** لإضافة وتعديل أخبار الجامعة.
- **الأسئلة الشائعة (`/api/faq`):** لإدارة قسم الأسئلة.
- **الحضور (`/api/attendance`):** لتسجيل حضور الطلاب.

*(يوجد أكثر من 25 ملف Route لتغطية كافة جوانب النظام الجامعي).*

## كيفية الإعداد (Configuration)
يجب إنشاء ملف `.env` في هذا المجلد يحتوي على المتغيرات التالية:
- `PORT`: المنفذ الذي سيعمل عليه الخادم (افتراضياً 5000).
- `DB_HOST`, `DB_USER`, `DB_PASS`, `DB_NAME`: بيانات الاتصال بقاعدة البيانات.
- `JWT_SECRET`: مفتاح السر لتشفير التوكينز.
- `FRONTEND_URL`: رابط الواجهة الأمامية للسماح بـ CORS.

## تشغيل الخادم
```bash
npm run dev # للتشغيل في وضع التطوير (باستخدام nodemon)
npm start   # للتشغيل في وضع الإنتاج
```
